API Documentation

Welcome to the Velo Backend API documentation. This API powers the Velo ride-hailing and courier platform, providing robust services for users, drivers, and merchants.

Base URL: /api/v1 Auth: x-api-key header

Authentication

POST /auth/request-otp
API Key Required

Requests a 6-digit verification code to be sent via SMS/WhatsApp to the provided phone number.

Request Body 
{
  "phoneNumber": "+233536690447"
}
Response (200 OK) 
{
  "message": "OTP sent successfully"
}
POST /auth/verify-otp
API Key Required

Verifies the OTP code received by the user and returns an access token.

Request Body 
{
  "phoneNumber": "+233536690447",
  "code": "123456"
}
Response (200 OK) 
{
  "user": { ... },
  "token": "jwt-token-string"
}
POST /auth/sync
API Key Required Auth Token Required

Synchronizes the authenticated user's data with the backend system.

Request 
{} // Provided via Authorization header
Response (200 OK) 
{
  "user": { ... },
  "synced": true
}

Profiles

POST /profile/buyer
Buyer Only Auth Required

Sets up or updates the buyer profile for the authenticated user.

Request Body 
{
  "fullName": "John Doe",
  "region": "Greater Accra",
  "primaryLocation": "Adabraka, Accra"
}
Response (200 OK) 
{
  "id": "uuid",
  "fullName": "John Doe",
  "role": "buyer"
}
POST /profile/driver
Driver Only Auth Required

Sets up the driver profile including vehicle and license details.

Request Body 
{
  "licenseNumber": "GHA-123456",
  "vehicleType": "bike",
  "vehicleModel": "Royal 125",
  "vehiclePlate": "GW-2022-21"
}
POST /profile/merchant
Merchant Only Auth Required

Sets up a merchant business profile.

Request Body 
{
  "businessName": "Velo Express",
  "category": "courier",
  "location": "Darkuman, Accra"
}

Rides

POST /rides/estimate
Buyer Only API Key Required

Gets fare estimates for all available vehicle types based on pickup and dropoff locations.

Request Body 
{
  "origin": { "lat": 5.6037, "lng": -0.1870 },
  "destination": { "lat": 5.6148, "lng": -0.2058 }
}
Response (200 OK) 
[
  {
    "vehicleType": "bike",
    "estimatedFare": 15.50,
    "distance": "3.2 km",
    "duration": "8 mins"
  },
  {
    "vehicleType": "standard",
    "estimatedFare": 35.00,
    "distance": "3.2 km",
    "duration": "12 mins"
  }
]
POST /rides/request
Buyer Only Auth Required

Requests a new ride. System will search for available drivers.

Request Body 
{
  "pickupAddress": "Airport",
  "pickupLat": 5.6037,
  "pickupLng": -0.1870,
  "dropoffAddress": "Osu",
  "dropoffLat": 5.6148,
  "dropoffLng": -0.2058,
  "vehicleType": "bike",
  "distanceKm": 3.2,
  "durationMin": 10
}
Response (201 Created) 
{
  "ride": {
    "id": "uuid",
    "status": "searching",
    "fare": 15.50
  }
}
POST /rides/:id/payment
Buyer Only Auth Required

Sets the payment method for a ride and initiates payment if applicable.

Request Body 
{
  "paymentMethod": "momo",
  "phoneNumber": "23354..."
}
POST /rides/:id/cancel
Buyer/Driver

Cancels a pending or active ride.

{ "reason": "Changed my mind" }
GET /rides/active
Buyer Only

Gets the current active ride for the authenticated buyer.

GET /rides/history
Buyer Only

Gets ride history with pagination (?limit=x&offset=y).

Driver Operations

POST /driver/online
Driver Only API Key Required

Sets the driver's status to online, making them eligible to receive ride requests.

Request 
{}
Response (200 OK) 
{
  "status": "online",
  "message": "You are now online"
}
POST /driver/location
Driver Only

Updates the driver's current real-time GPS coordinates.

Request Body 
{
  "lat": 5.6037,
  "lng": -0.1870,
  "heading": 180,
  "speed": 15
}
Response (200 OK) 
{ "message": "Location updated" }
POST /driver/online
Driver Only

Sets the driver's status to online. Requires current lat/lng.

{ "lat": 5.60, "lng": -0.18 }
POST /driver/offline
Driver Only

Sets the driver status to offline.

POST /driver/rides/:id/accept
Driver Only

Accepts an incoming ride request.

{ "driverName": "Kwame" }
POST /driver/rides/:id/start
Driver Only

Marks the ride as started (passenger picked up).

POST /driver/rides/:id/complete
Driver Only

Completes the ride and triggers payment processing.

GET /driver/stats
Driver Only

Gets driver performance stats (rating, earnings, etc.).

Payments

POST /payments/webhook
Unprotected

Webhook endpoint for Paystack notifications.

POST /payments/verify/:reference
Auth Required

Manually verify a payment reference.

Wallet

GET /wallet

Gets the authenticated user's wallet balance.

GET /wallet/transactions

Gets wallet transaction history.

Saved Locations

POST /locations
Buyer Only

Saves a personal location (Home, Work, etc.).

{ "label": "Home", "address": "...", "lat": 5.6, "lng": -0.1 }
GET /locations

Gets all saved locations for the user.

Ratings

POST /ratings

Submit a rating for a completed trip.

{ "rideId": "uuid", "rating": 5, "comment": "Great ride!" }

Places & Search

GET /places/autocomplete

Search for places as you type (?input=...).

POST /places/reverse-geocode

Get address from lat/lng coordinates.

{ "lat": 5.6037, "lng": -0.1870 }

Notifications

GET /notifications

Fetch user notification history.

POST /notifications/push-token

Registers a push notification token (Expo/FCM).

{ "token": "...", "platform": "ios" }
