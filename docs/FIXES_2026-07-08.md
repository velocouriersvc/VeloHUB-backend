# Fixes - 2026-07-08

## 1. Chat showed hardcoded "Benjamin Kojo"  [FIXED, app]
`DriverChatModal` hardcoded the recipient name. Now takes a `driverName` prop (header
+ placeholder) fed from `activeRide.driver.driverProfile.fullName` in `rides.tsx`.

## 2 + 5. "I've arrived" does nothing / live tracking never advances  [FIXED]
Root cause: the ride lifecycle guards were too strict - `driverEnroute` required
`status = PAID` and `driverArrived` required `DRIVER_ENROUTE`. When payment had not
settled, the driver's actions threw and the customer's status/tracking never moved.
- Backend (`ride-service.ts`): relaxed `driverEnroute` / `driverArrived` / `startRide`
  to allow forward progress from any active pre-trip state. Payment settles
  independently (cash on completion, momo/card via webhook). Pushed (`a7dba9f`).
- App (`navigation.tsx`): the trip CTA buttons now actually call the backend
  (arrived / start / complete) and signal `enroute` on open, instead of only changing
  local state.

## 5. Driver car marker does not move  [FIXED, app]
The driver only streamed GPS from the full-screen map. Now the active-trip screen
(`(driver-tabs)/rides.tsx`) also streams location (`emitLocation`) while a ride is
active, so the customer's marker moves without opening the map.

## 3. Map does not direct the driver to the destination  [SETUP VERIFIED]
`navigation.tsx` already draws `MapViewDirections` from the driver to the pickup, then
to the dropoff when the trip starts, using the ride's coordinates (which the backend
returns). With the lifecycle now advancing (item 2), the route switches to the dropoff
when the ride starts. Needs a fresh EAS build to confirm on device.

## 6. "Report a problem" button did nothing  [FIXED, app]
`app/orders/track.tsx`: the button now opens a support prompt (call / email
support@velocouriersvc.com, with the ride/order ref in the subject).

## 8. New Nigeria pricing + Velo Priority tier + booking fee  [DONE end-to-end]
- Backend: new `VehicleType.PRIORITY`, a `bookingFee` column, and
  `fare = base + distance + time + booking fee`. Seed applies EXACT client NG fares
  (no global +20%): bikes 500/95/20 (+200), basic 1000/150/30 (+350), priority ~basic
  x1.25 (client 1.16-1.35 range), premium 1200/210/35 (+500), trucks 2500/265/35 (+250).
  A Velo Priority tier is added per country (car x1.25). Idempotent enum + column
  auto-migration. Pushed (`44e5c9c`).
- App: added the Velo Priority tier + image (`velo-priority.jpg`), backend-type mapping
  (`priority`), and vehicleType unions.
- Admin: fee page shows/edits the Booking Fee and the Priority tier. Committed
  (`83d3df4`) and DEPLOYED to the VPS.

## 4 + 7. Payment prompt for booking / scheduling  [PARTIAL]
