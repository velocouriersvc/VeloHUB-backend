# Fixes - 2026-07-09

Batch addressing the second round of app issues, delivered in vertical slices (each
typechecked; backend 63/63 tests green). Commission was intentionally left at 15/85 per
the user's later instruction (an earlier 75/25 change was reverted).

## Driver "I've arrived" stuck on En Route  [FIXED]
Root cause: the driver nav screen ran a local step machine that never read the real ride
status, and the backend guards were status-exact, so any re-tap/out-of-order tap threw
"Ride is not in a state to mark arrived".
- Backend `ride-service.ts`: lifecycle transitions (enroute/arrived/start/complete) are
  now rank-based and idempotent - already-in-or-past target is a safe no-op.
- App `navigation.tsx`: seeds the step from `ride.status`, only emits enroute pre-arrival.
- `_layout.tsx`: hide the floating driver tab bar on trip screens (navigation/rides/
  deliveries) so it no longer overlaps the "I've arrived" CTA.

## Right driver + customer info everywhere  [FIXED]
- Backend: `getDriverActiveRide` loads the customer relation; the driver API returns real
  `customerName`/`customerPhone` (raw customer + `pickupCode` stripped).
- Customer ride payload carries real `driverStats` (avg rating + completed trips);
  `RideTracking` shows them instead of the hardcoded 4.9 / "1,242 rides".
- Driver nav reads correct `finalFare`/`distanceKm`/`durationMin`.
- `remove-driver.ts`: guarded ops script (dry-run by default) to delete a stale test
  driver account (e.g. Benjamin Kojo). Run with `CONFIRM_DELETE=true`.

## Lydia (NG driver) not receiving requests  [FIXED]
There is no country filter in matching. Two real causes fixed:
- Exact `vehicleType` match dropped requests once NG had a unique `priority` tier -> now a
  tier-compatibility map (car <-> priority).
- Idle online drivers stopped refreshing their Redis location because the sync read a
  stale `isOnline` closure (always false) -> now reads `isOnlineRef`, so they don't expire
  out of matching after 5 min.

## In-app chat, end to end  [FIXED]
Was a local-only mock. Now: `RideMessage` model + `RideMessageService`, socket relay in
both namespaces (customer <-> driver), `GET /rides/:id/messages` history. Customer and
driver screens send/receive live and load history.

## Pickup-screen scrolling  [FIXED]
The customer sheet wrapped everything in one pan gesture. Now the pan is attached to a
drag handle only, so the inner ScrollViews scroll freely.

## Live tracking + real ETA  [FIXED]
Customer subscribes to the ride-scoped `driver:location` stream (was ignored) and drives
the marker from it. Real route ETA from MapViewDirections replaces the hardcoded
"3 mins"/"12 mins".

## Ride pickup confirmation code  [FIXED]
Opt-in at booking (off by default): `Ride.requireCode` + `pickupCode`; generated when the
rider toggles it. `startRide` is gated on the code. Customer sees a shareable code card;
the driver enters it to start (cross-platform modal). Product pickups keep their existing
codes.

## Global scaling  [FIXED]
- App: `isRideAndDeliveryCountry` / `isServicesProviderCountry` accept any country.
- Backend: dropped the Africa-only deactivation; all markets active + a global DEFAULT
  (USD) `platform_settings` row; `fare-service` falls back to US pricing + DEFAULT/US
  settings so fares resolve worldwide; payment registry defaults unknown countries to
  Stripe (Paystack kept for GH/NG/KE/ZA mobile money).

## Priority image + 20 km radius  [FIXED]
- Priority tier uses the Standard/Basic image (`velo-basic.png`).
- Driver search radius set to 20 km; merchant/search fallbacks 10 -> 20 km.

## Turn-by-turn voice directions (accessibility)  [FIXED]
Added `expo-speech`; the nav screen captures route steps and speaks the next maneuver as
the driver approaches (70 m). Toggle in the top bar, off by default. Needs a fresh EAS
build (native dep).

## Ride-request tone  [VERIFIED - already reliable]
Foreground handler plays sound on any screen; the `rides` Android channel is HIGH
importance + sound for backgrounded requests; the Jobs screen adds a looping alert that
stops on accept/decline/timeout. No change required.
