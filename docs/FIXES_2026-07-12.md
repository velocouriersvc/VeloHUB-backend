# Fixes - 2026-07-12 (round 5)

## Chat + live navigation STILL failing on the new build - the real root cause  [FIXED]
Production runs velo-api with replicas: 2 and there was NO socket.io Redis adapter.
Socket rooms do not span pods, so whenever the customer and driver landed on different
pods, chat relay and the ride-room driver:location stream silently vanished. (The earlier
single-pod verification passed because both test clients hit one pod.)
- Added @socket.io/redis-adapter wired to the existing ioredis connection - rooms and
  emits now work across all replicas.
- k8s/api.yaml NodePort service gains sessionAffinity: ClientIP (protects the polling
  fallback transport). Apply with kubectl on the VPS.
- Bonus: socket location:update now refreshes the driver's Redis status TTL (value
  preserved - busy stays busy), so streaming drivers never silently expire out of
  matching after 5 minutes.

## Messaging is now persistent by construction  [FIXED]
Sends previously went over the socket only: with a broken room, the optimistic bubble
showed and nothing persisted - reopening chat lost everything.
- New POST /rides/:id/messages (buyer or driver): persists, relays to both socket rooms,
  and push-notifies the recipient - one shared delivery path used by REST and sockets.
- Both apps now SEND via REST (optimistic bubble; server row replaces it; failure removes
  it with an alert) while the socket stays the live receive channel.
- History loads as soon as the ride is known (not only when the chat opens) and MERGES
  with in-memory messages instead of replacing them.

## Customer live map  [FIXED]
RideTracking refused to render a map at all until the driver streamed a position. The
map region now falls back to pickup/destination, the pickup->destination route draws
immediately, and the driver marker/LIVE badge appear when the stream arrives.

## "Only Pay with Card" + no MoMo prompt  [FIXED]
- The method menu filtered on the DEVICE-geo country (often null/late) - it now uses the
  user's ACCOUNT country, so African accounts always see Mobile Money + Cash + Card.
- The charge itself dropped `country` (hardcoded GH/GHS/Paystack). The app now sends the
  account country through setPayment -> ride-service -> payment-service.
- MoMo rewritten to Paystack's HOSTED checkout restricted to the mobile_money channel:
  the app opens the returned authorization_url and Paystack drives the telco prompt/OTP
  end to end (the old raw /charge flow ignored pay_offline/send_otp/display_text and
  returned no URL, so no prompt ever reached the phone). Webhook flow unchanged.

## "Drivers available within 3 mins" + "No drivers"  [FIXED]
- The home card was a hardcoded slogan. It now shows the live feed: "N drivers near you -
  closest ~X min" or "No drivers nearby right now" (coords the screen already fetched).
- Overlay radii aligned with matching (10 -> 20 km) on the socket query, REST endpoint,
  and app default, so the vehicle list, the map overlay, and actual booking agree.
- The biggest cause of false "No drivers" was the status-TTL expiry fixed above.

## New fares (GH / USA / CANADA / NIGERIA) - admin-configurable end to end  [DONE]
- vehicle_pricing gains bookingFeePercent (% of base+distance+time) and roadLevy (flat);
  zones gains flatSurcharge. All added via idempotent column migrations.
- computeRideFare folds flat fee + percent fee + road levy + zone surcharge into the
  fare subtotal (single formula, unit-tested path).
- Exact price books applied (NO legacy +20% for GH/US/CA/NG): GH per spec; USA per-mile
  converted to per-km (/1.60934); Canada per-km as given; Nigeria with 5% platform
  booking fee + 20 road levy. Velo Priority = Basic x1.25 (client range 1.16-1.35).
  minimumFare = base fare so the new small fees are never swallowed by the floor.
- Geofence surcharges: zones are now WIRED INTO the fare path (haversine pickup/dropoff
  in-radius check) for estimates AND ride creation; estimate API accepts dropoff coords
  and the app sends them. Seeded NG zones: LOS Airport Dropoff (+1500), Lekki-Ikoyi Link
  Bridge (+400) - find-or-create, admin-editable in Zones.
- SEED SAFETY: pricing seeding is now VERSIONED (seed_meta table). The full overwrite
  runs only when PRICING_SEED_VERSION increases; ordinary boots only insert missing
  rows - admin dashboard edits finally survive restarts. The Africa-only deactivation
  was removed (global operation).
- Admin: FeeStructure edits Booking Fee % + Road Levy per vehicle row; Zones edits the
  Flat Surcharge. Backend admin update type extended.

## Dual gateway + Stripe Identity  [DONE]
- Registry now routes ALL African countries to Paystack; everything else defaults to
  Stripe (was only GH/NG/KE/ZA).
- Stripe Identity: the stubbed GET /auth/identity-status is real now (reads the persisted
  Identification via the user's profile links); the existing StripeIdentityCard is
  surfaced on the customer profile for non-African accounts (US/CA/EU), wired to the
  session + live status. Requires STRIPE_SECRET_KEY in the pod env.
