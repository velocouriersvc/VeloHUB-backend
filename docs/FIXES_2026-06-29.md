# Fixes - 2026-06-29 (user-reported issues)

## 1. Product/checkout delivery address does not autocomplete (ride works)
Cause: the checkout delivery/pickup/stop suggestion dropdowns used a `FlatList`
nested inside the checkout `ScrollView` (a nested VirtualizedList), which fails
to render on Android. The ride picker is a full-screen list, so it worked.
Fix: render suggestions with a plain `.map()` in a `View`.
File: `VeloHUB/app/checkout.tsx` (removed the now-unused `FlatList` import).

## 2. Order says "ready, looking for a driver" on payment / should be seller-driven
The backend already keeps a new order at PENDING ("awaiting confirmation from the
merchant") and does NOT auto-advance it; only the merchant moves it
ACCEPTED -> PREPARING -> READY, and only READY_FOR_DELIVERY makes it available to
drivers. The premature "Order Prepared" prompt comes from an older deployed build.
Added: the seller now also gets an **SMS** on every new order (best-effort) so they
can accept and decide readiness even with the app closed.
File: `VeloHUB-backend/src/services/order-service.ts`.
Note: the "package returned ... did.li/..." SMS is an EXTERNAL phishing scam, not
sent by VeloHUB (our SMS go via Prelude with the VeloHUB sender id).

## 3. Images show on iOS but not Android
Causes covered: cleartext/wrong-host image URLs that Android blocks but iOS tolerates.
Fixes:
- App: `android.usesCleartextTraffic = true` (`VeloHUB/app.json`).
- Backend: product image URLs are normalized to the public https asset host on every
  read path (`toPublicImages` in `product-service.ts`).
If images are HEIC or the host's TLS chain is incomplete, that is a separate
server/cert item to verify (Android is stricter than iOS on both).

## 4. Launch icon is blue on Android (white on iOS)
Cause: `android.adaptiveIcon.backgroundColor` was `#0D78C4` (blue).
Fix: set to `#FFFFFF` (`VeloHUB/app.json`).

## 5. Still unable to set up a Driver Profile
- Notifications during profile setup are best-effort (a notification failure can no
  longer 500 a committed setup) and the `notification_type_enum` is synced on boot.
- Added defensive truncation of driver fields to their column lengths so a long
  value (e.g. a full address in `region` varchar(100)) cannot overflow and 500.
File: `VeloHUB-backend/src/services/profile-service.ts`.
These backend fixes require deploy (merge develop -> main).
